ES6之Proxy使用

什么是proxy？

在es2015中，Proxy是为对象设置访问代理器（代理如同门卫），它可以轻松监视到对象的读写过程。

proxy的应用场景示例

在es2015之前，如果我们想要监视某个对象的属性读写，那我们可以使用es5 提供的Object.defineProperty方法来为我们的对象添加属性，捕获到对象属性的读写过程（Vue3.0之前 就是使用这个方法实现数据响应，从而实现数据双向绑定）。

那我们如何使用Proxy？

  • new Proxy(需要代理的目标对象， 代理的处理对象);
  • 首先抛出示例代码

都知道在Vue3.0中使用proxy替代了Object.defineProperty(),那相对于 Object.defineProperty()，proxy有什么优势呢？

proxy的优势

Proxy更强大一些：defineProperty 只能监视对象属性的读写，而Proxy能够监视到更多对象操作（比如说delete 操作，对对象方法的调用等等）。

举个栗子

const list = [];

const listProxy = new Proxy(list, {
set(target,property, value) {
console.log(‘set’,property, value);
target[property] = value;
return true; // 表示设置成功
}
});

listProxy.push(100); //set 0 100 (0 是数组下标，100 是数组下标对应的值)

const person = {}

Object.defineProperty(person, ‘name’, {
get() {
console.log(‘name 被访问’);
return person._name
},
set (value) {
console.log(‘name 被设置’);
person._name = value;
}
});

Object.defineProperty(person, ‘age’, {
get() {
console.log(‘age 被访问’);
return person._age
},
set (value) {
console.log(‘age 被设置’);
person._age = value;
}
});

person.name = ‘jack’;
console.log(person.name);
//name 被设置
// name 被访问
// jack
// jack

// Proxy 方式更合理
const person2 = {
name: ‘xh’,
age: 20
};

const personProxy = new Proxy(person2, {
get(target, property){
return target[property];
},
set (target, property, value) {
target[property] = value;
return true;
}
});

personProxy.name = ‘jack’;
console.log(personProxy.name); // jack
````
